# Tabular Foundation Models: From Classical Methods to Modern Foundation Models

---

## 핵심 비교 (Overview)

| 모델 | 등장 배경 | 핵심 아이디어 | 수식 핵심 | 강점 | 한계 |
|------|----------|-------------|----------|------|------|
| Tree (XGBoost) | feature engineering 한계 | additive tree ensemble | $\sum f_m(x)$ | 강력한 baseline | representation 없음 |
| TabTransformer | categorical 처리 문제 | feature embedding + attention | Transformer(H) | categorical interaction | numeric 약함 |
| FT-Transformer | unified modeling 필요 | 모든 feature를 token화 | Attention(Q,K,V) | general framework | 데이터 요구 큼 |
| SAINT | dataset 구조 무시 문제 | row + column attention | dual attention | dataset-aware | 복잡도 증가 |
| TabPFN | small data + tuning 문제 | Bayesian inference 근사 | $P(y|x,\mathcal{D})$ | small data SOTA | scale 제한 |
| TabSTAR | foundation 방향 필요 | target-aware learning | $f(x,t)$ | semantic 이해 | 초기 연구 |

---

## 1. 왜 Tabular Foundation Model이 필요한가

Tabular 데이터는 실제 산업에서 가장 많이 사용되지만, paradoxically 가장 어렵다.

그 이유는 다음과 같다.

- feature 간 구조가 없음 (이미지처럼 spatial 구조 없음)
- column마다 의미가 완전히 다름 (heterogeneous)
- 데이터 수가 적음 (small data regime)
- missing, imbalance, noise 존재

따라서 기존 deep learning이 잘 작동하던 전제:

> large + structured + homogeneous

이 깨진다.

이 문제를 해결하기 위해 발전한 흐름은 다음과 같다.

> Tree → Deep Tabular → Transformer → Meta-learning → Foundation Model

이 흐름을 이해하는 것이 중요하다. 각각은 이전 방법의 한계를 해결하기 위해 등장한다.

---

## 2. 전통적 접근: Tree-based Models

가장 강력한 baseline은 여전히 gradient boosting이다.

$$
\hat{y} = \sum_{m=1}^{M} f_m(x)
$$

각 항의 의미는 다음과 같다.

- $f_m$: $m$번째 decision tree
- $M$: tree 개수
- $\hat{y}$: 최종 예측값

이 구조의 의미는 다음과 같다.

- 각 tree는 이전 tree의 residual을 보정
- feature interaction을 자동으로 학습

하지만 중요한 한계가 있다.

- representation learning이 없음
- dataset 간 transfer 불가능
- foundation model로 확장 불가능

이 한계를 해결하려는 시도가 deep tabular 모델이다.

---

## 3. Transformer 기반 Tabular Learning

### 3.1 왜 Transformer가 필요한가

이미지와 NLP에서 성공한 이유는 다음과 같다.

- token 간 관계를 attention으로 학습
- long-range dependency modeling

이를 tabular에 적용하면:

> "feature 간 관계를 학습할 수 있지 않을까?"

라는 아이디어가 등장한다.

---

## 4. TabTransformer

### 4.1 핵심 문제

범주형 변수는 one-hot encoding으로는 정보 손실이 크다.

### 4.2 해결 방법

각 column을 embedding으로 변환한다.

$$
e_i = \text{Embedding}(x_i^{cat})
$$

- $x_i^{cat}$: $i$번째 categorical feature
- $e_i$: embedding vector

전체 feature는 다음과 같이 구성된다.

$$
H = [e_1, e_2, \dots, e_k]
$$

Transformer 적용:

$$
H' = \text{Transformer}(H)
$$

출력:

$$
\hat{y} = g(H')
$$

### 4.3 왜 중요한가

이 모델은 처음으로 다음을 가능하게 했다.

- categorical feature 간 interaction 학습
- column 관계 modeling

---

## 5. FT-Transformer

### 5.1 왜 등장했는가

TabTransformer는 categorical 중심이었다.

하지만 실제 데이터는:

- numeric + categorical 혼합

### 5.2 핵심 아이디어

모든 feature를 token으로 통일한다.

numeric 처리:

$$
z_i = w_i x_i^{num}
$$

- $x_i^{num}$: numeric feature
- $w_i$: learnable weight

전체 입력:

$$
H = [z^{num}, e^{cat}]
$$

Attention:

$$
\text{Attention}(Q,K,V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d}}\right)V
$$

각 항 의미:

- $Q$: query (현재 feature)
- $K$: key (다른 feature)
- $V$: value (정보 전달)
- $d$: scaling factor

### 5.3 의미

- 모든 feature를 동일한 방식으로 처리
- feature interaction 자동 학습

---

## 6. SAINT

### 6.1 왜 등장했는가

기존 모델은 feature만 보고 있었다.

하지만 실제 dataset에는:

- sample 간 관계도 존재

### 6.2 핵심 구조

column attention:

$$
H_c = \text{Attention}(features)
$$

row attention:

$$
H_r = \text{Attention}(samples)
$$

### 6.3 의미

- column → feature interaction
- row → sample similarity

즉 dataset 구조 자체를 학습한다.

---

## 7. TabPFN (핵심 전환점)

### 7.1 왜 등장했는가

Transformer 기반 모델의 한계:

- 데이터 많이 필요
- tuning 필요
- small data에서 불안정

### 7.2 핵심 아이디어

> "학습이 아니라 Bayesian inference를 직접 학습하자"

우리가 원하는 것은:

$$
P(y \mid x, \mathcal{D})
$$

Bayesian 형태:

$$
P(y \mid x, \mathcal{D}) = \int P(y \mid x, \theta) P(\theta \mid \mathcal{D}) d\theta
$$

각 항의 의미:

- $P(y \mid x, \theta)$: 모델 예측
- $P(\theta \mid \mathcal{D})$: posterior
- 적분: 모든 모델 평균

### 7.3 TabPFN 방식

이 적분을 neural network로 근사:

$$
\hat{y} = f_\theta(\mathcal{D}, x)
$$

핵심 차이:

- 기존: $f(x)$
- TabPFN: $f(\mathcal{D}, x)$

### 7.4 의미

- dataset 자체가 입력
- tuning 없이 inference 가능
- small data 최적화

---

## 8. TabSTAR (Foundation 방향)

### 8.1 왜 등장했는가

TabPFN의 한계:

- task generalization 제한
- semantic 이해 부족

### 8.2 핵심 아이디어

> "target-aware representation"

수식:

$$
z = f(x, t)
$$

각 항 의미:

- $x$: input feature
- $t$: target context (label semantics)
- $z$: representation

### 8.3 의미

- 단순 feature가 아니라 “의미 기반 학습”
- LLM과의 연결 가능

---

## 9. 통합 관점

모든 tabular foundation model을 하나로 정리하면:

$$
z = f_\theta(x, \mathcal{D}, t)
$$

$$
\hat{y} = g(z)
$$

각 항 의미:

- $x$: input sample
- $\mathcal{D}$: dataset context
- $t$: target context
- $z$: representation
- $\hat{y}$: prediction

---

## 10. 흐름 요약 (중요)

전체 흐름은 다음과 같다.

1. Tree-based methods  
   → 강력하지만 representation 없음  

2. Transformer-based models  
   → feature interaction 학습  

3. Dataset-aware models (SAINT)  
   → dataset 구조 반영  

4. Meta-learning (TabPFN)  
   → dataset 자체를 학습  

5. Foundation model (TabSTAR)  
   → semantic + generalization
